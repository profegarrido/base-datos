# Introducción a Bases de Datos

¡Bienvenidos al curso de Introducción a Bases de Datos! 

Este curso está diseñado para proporcionar una base sólida tanto en el modelado teórico como en la implementación práctica de sistemas de gestión de datos modernos, priorizando herramientas ligeras y portátiles.

En este repositorio, encontrarán todos los materiales necesarios para la asignatura:

- **Talleres de Modelado:** Resolución de casos reales para diseño conceptual.

- **Laboratorio de Consultas:** Ejercicios prácticos de SQL y NoSQL en entornos locales y portátiles.

- **Guías Teóricas:** Documentación sobre álgebra relacional, normalización y ecosistemas de datos.

## Temario
El curso se divide en cuatro unidades principales, siguiendo una metodología de Aprendizaje Basado en Problemas (ABP):

### 1. Modelado Conceptual y Lógico
Diseño Conceptual: Entidades, atributos, relaciones y cardinalidad.

Herramientas de Modelado: Uso de diagramas para casos de estudio institucionales.

Diseño Lógico: El modelo relacional, transformación de entidades a tablas y gestión de claves (PK/FK).

Restricciones de Integridad: Dominios y reglas del modelo.

### 2. Fundamentos Matemáticos y Normalización
Teoría de Conjuntos y Álgebra Relacional: Operaciones de selección, proyección, unión y joins.

Normalización: Aplicación de formas normales (1FN hasta BCNF).

Optimización: Control de anomalías y eficiencia en la estructura de datos.

### 3. SQL y Manipulación de Datos (Enfoque Portable)
Lenguaje SQL Estándar: Uso de DDL (Definición), DQL (Consultas) y DML (Manipulación).

Operaciones CRUD: Gestión avanzada de datos, agrupamiento y ordenamiento.

Implementación con SQLite: Creación y gestión de bases de datos mediante archivos locales .db.

### 4. Ecosistemas NoSQL y Datos No Estructurados
Modelos NoSQL: Teorema CAP, consistencia eventual vs. ACID.

Tipologías: Documentos (JSON/BSON), clave-valor, columnas y grafos.

Preprocesamiento: Limpieza y transformación de datos no estructurados.

Análisis Crítico: Debates técnicos sobre cuándo usar bases de datos embebidas vs. cliente-servidor.
